Lab 6: Forms

Lab 6: Forms - create the Grade Store web page

The Grade Store page sells A+, A, B+, and B grades for various courses that students can buy with their credit card. (Just a joke!)

The Grade Store page (gradestore.html) has a form that submits to cheater.php. Then cheater.php will output a web page to confirm that the student's money was taken out.

Lab 6 exercises

Download the gradestore.zip and complete the following exercises.

  1. Create a Form
  2. Process Form Data
  3. Save Form Data
  4. Basic Data Validation
  5. Name & Credit Card Check

Ex 1: Create a Form

(See example screenshot on next slide.)

Create an HTML form that POSTs its submitted data to a PHP program on a server.

  • Modify the gradestore.html by turning part of its contents into an HTML form:
    • create two text fields for Name and ID
    • create checkboxes for courses - CSE326, CSE107, CSE603, CIN870
    • create a drop-down list for grades - A+, A, B+, B
    • create a text field for Credit Card and radio buttons for - Visa, MasterCard
    • create a submit button with the button text - I am a giant cheater!
  • Note :
    • name attribute must be given to form controls for the data to be sent as query parameters
    • some form controls requires value attribute

Ex 2: Process Form Data

(See example screenshot on next slide.)

Write the PHP page to handle the submitted form data.

  • Modify gradestore.html to POST its data to cheater.php.
  • Modify the cheater.php to display submitted data.
    • data to be displayed - name, id, selected courses, selected grade, credit card number and type
    • for now, the page does not need to save this information on the server
    • do NOT use print or echo in PHP: use expression block
  • Complete the PHP function processCheckbox :
    • processes a given array of names (values of name attributes for all the checkboxes)
    • return a single string of all the selected checkboxes' names with comma separation (, )

Ex 3: Save Form Data

(See example screenshot on next slide.)

Modify cheater.php to save the submitted data to a file loosers.txt.

  • Use the following format:

  • Change your page's output to show the complete contents of loosers.txt.
    • place the file contents into an HTML <pre> element to preserve whitespace
  • Use file_get_contents and file_put_contents.

Ex 4: Basic Data Validation

(See example screenshot on next slide.)

Update cheater.php to verify that all parameters exist and are not blank.

  • Check the existence of each parameter using the PHP function isset.
  • Check the blankness of an element in $_POST by comparing it to the empty string.
  • If the form is filled incorrectly, show an error message as seen on the next page.
    • provide a Try again link that directs back to the initial gradesotre.html page

Ex 5: Name & Credit Card Check

(See example screenshot on next slide.)

Update cheater.php to further validite the name and the credit card number.

  • Check if the name is composed of alphabets, dash(-), or a single white space.
  • Check if the credit card number is composed of exactly 16 digits.
  • Check if the Visa card starts with 4 and MasterCard starts with 5.
  • If the name or credit card number does not pass the above checks, show an error message as seen on the next page.
